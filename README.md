# WritingWorkbench

长篇写作结构工作台，一个面向网文作者和长篇小说作者的本地写作原型。

它不是 AI 代写器，也不是普通码字软件。它的核心目标是降低长篇创作里的认知摩擦：让作者把章节、正文、灵感卡片、节奏标记、统计工具和基础排版放在同一个工作流里。

## 项目定位

WritingWorkbench 更像一个小说创作工作台，而不是一个单纯的文档编辑器。

长篇写作往往不是线性发生的。作者可能先想到一句对白、一个画面、一个反转、一个情绪，再慢慢把它们整理成章节和结构。这个项目的第一版围绕这种真实写作方式设计：

```text
首页书架 -> 进入小说 -> 写章节 -> 拉出灵感池 -> 整理/关联灵感卡
```

## 当前功能

- 首页书架：展示小说项目、书名、字数和最近编辑时间。
- 章节编辑：创建章节、分卷、编辑正文、自动保存。
- 节奏标记：支持 `空`、`-`、`--`、`+`、`++`、`!`，用于记录章节情绪走势。
- 灵感池：支持卡片墙和便签流，新建、编辑、删除、拖动排序灵感卡。
- 灵感关联：一个灵感卡可以关联多个章节，也可以不关联任何章节。
- 码字统计：章节字数、本次码字、活跃码字时速、本次时长。
- 常规工具：导入、导出、小黑屋、历史版本、全文搜索等原型功能。
- 排版设置：中文字体、字号、行距、编辑区宽度、背景和一键排版选项。
- 夜间模式：支持日间/夜间切换。

## 产品原则

- 作者主权优先：AI 只能做摘要、检索、压缩和辅助，不替作者写故事。
- 低摩擦交互：能少点一次就少点一次，尽量减少写作时的窗口切换。
- 先接住混沌：灵感池允许碎片、对白、画面和设定先随手放进去。
- 结构服务写作：章节、灵感、节奏和工具应该围绕正文创作展开。
- 克制功能堆叠：不为了“看起来强大”加入会分散作者注意力的功能。

## 如何运行

这是一个本地静态网页原型。

在 Windows 上可以双击：

```text
打开写作台.bat
```

然后浏览器会打开：

```text
http://127.0.0.1:4173/index.html
```

也可以直接打开 `index.html`，但推荐使用启动脚本，因为本地服务方式更接近真实使用环境。

## 数据说明

当前版本使用浏览器本地存储保存数据。它适合原型验证和个人试用，但还不是正式生产级存稿工具。

重要稿件请自行备份。

## 后续方向

- AI 辅助章节摘要，而不是 AI 续写。
- 更稳的本地文件保存。
- 更清晰的版本管理。
- 更成熟的搜索和前文回顾。
- 更适合网文作者的节奏分析。
